## Overview

Add a Password Management section to the existing Telefutura CRM. This section stores and displays login credentials for telecom brand portals (WindTre, Vodafone, Sky, Fastweb, Energia) with a 4-step filtering flow.

---

## Requirements Summary

### User Flow (4 Steps)
1. **Brand Selection** — User clicks brand card (WindTre, Vodafone, Sky, Fastweb, Energia)
2. **Category Selection** — User selects category specific to brand
3. **Store Selection** — User selects store (Roma Termini, Milano Centrale, etc.)
4. **Password Table** — Display filtered credentials table

### Access Control
- **Roles with access:** `admin`, `store_manager`
- **Roles without access:** `agente`
- Admin can add/edit/delete credentials
- Store manager can only view

## Data Structure

### Table Columns
1. **TIPO DI ACCESSO** (Access Type) — e.g., "NGPOS Portal", "Dealer Dashboard"
2. **USERNAME** — login username with copy button
3. **PASSWORD** — masked by default, click eye icon to reveal

### Database Schema

```sql
CREATE TABLE password_credentials (
  id SERIAL PRIMARY KEY,
  brand_id VARCHAR(50) NOT NULL,          -- 'windtre', 'vodafone', 'sky', 'fastweb', 'energia'
  category_id VARCHAR(50) NOT NULL,        -- 'ngpos', 'ask', 'findomestic', etc.
  store_id VARCHAR(50) NOT NULL,           -- 'roma-termini', 'milano-centrale', 'tutti'
  access_type VARCHAR(255) NOT NULL,       -- Display name like "NGPOS Portal"
  username VARCHAR(255) NOT NULL,
  password_encrypted TEXT NOT NULL,        -- AES-256 encrypted
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  created_by INTEGER REFERENCES users(id),
  updated_by INTEGER REFERENCES users(id)
);

-- Index for fast filtering
CREATE INDEX idx_password_filter 
ON password_credentials(brand_id, category_id, store_id);

-- Audit log table
CREATE TABLE password_access_log (
  id SERIAL PRIMARY KEY,
  credential_id INTEGER REFERENCES password_credentials(id),
  user_id INTEGER REFERENCES users(id),
  action VARCHAR(50) NOT NULL,             -- 'view', 'reveal', 'copy'
  accessed_at TIMESTAMP DEFAULT NOW()
);
```

## API Endpoints

### Read Endpoints

```
GET /api/passwords/brands
Response: [{ id: 'windtre', name: 'WindTre', categoryCount: 4 }, ...]

GET /api/passwords/categories/:brandId
Response: [{ id: 'ngpos', name: 'NGPOS' }, ...]

GET /api/passwords/stores
Response: [{ id: 'roma-termini', name: 'Roma Termini' }, ...]

GET /api/passwords/credentials?brandId=windtre&categoryId=ngpos&storeId=roma-termini
Response: [
  {
    id: 1,
    accessType: "NGPOS Portal",
    username: "wind_ngpos_roma",
    password: "••••••••••"  // Always masked in response
  },
  ...
]
```

### Write Endpoints (Admin Only)

```
POST /api/passwords/credentials
Body: { brandId, categoryId, storeId, accessType, username, password }
Response: { id, ...credential }

PUT /api/passwords/credentials/:id
Body: { accessType?, username?, password? }
Response: { id, ...credential }

DELETE /api/passwords/credentials/:id
Response: { success: true }
```

### Password Reveal Endpoint

```
POST /api/passwords/credentials/:id/reveal
Response: { password: "actual_password" }
// This should log to password_access_log with action='reveal'
```

## Security Requirements

### Password Storage
- **Encrypt at rest** using AES-256
- Use environment variable for encryption key: `PASSWORD_ENCRYPTION_KEY`
- Never log or expose raw passwords in API responses
- Mask passwords as `••••••••` in all GET responses

### Access Logging
- Log every password reveal action
- Log every copy action (if trackable)
- Store in `password_access_log` table

### Rate Limiting
- Limit password reveal attempts: 10 per minute per user
- Return 429 Too Many Requests if exceeded

### Frontend Security
- Passwords hidden by default (masked dots)
- Eye icon to reveal (triggers API call)
- Copy button for username (always available)
- Copy button for password (only when revealed)

## UI/UX Specifications

### Visual Style
- Match existing CRM design (dark theme, glass-card effects)
- Brand-specific color coding:
  - WindTre: Orange (`#f97316`)
  - Vodafone: Red (`#ef4444`)
  - Sky: Sky Blue (`#0ea5e9`)
  - Fastweb: Violet (`#8b5cf6`)
  - Energia: Emerald (`#10b981`)

### Brand Selection (Step 1)
- 5 cards in responsive grid
- Each card shows:
  - Brand icon (Lucide icons)
  - Brand name
  - Category count badge
- Hover effect: scale 1.02, colored ring

### Category Selection (Step 2)
- Cards similar to brand selection
- Category name + credential count
- Back button to return to brand selection

### Store Selection (Step 3)
- List of store cards
- Store name + credential count
- "Tutti (Accesso Globale)" option for global credentials
- Back button to return to category selection

### Password Table (Step 4)
- Glass-card container
- Breadcrumb: `Brand > Category > Store`
- Table with 3 columns:
  1. TIPO DI ACCESSO
  2. USERNAME (with copy icon)
  3. PASSWORD (masked, with eye icon to reveal)
- Copy button: copies to clipboard, shows checkmark confirmation
- Reveal button: calls `/reveal` API, shows password temporarily
- Back button to return to store selection

### Security Notice
- Prominent amber-colored banner at bottom:
  - "Nota di sicurezza"
  - Warning about sensitive credentials
  - Reminder: limited access, don't share
